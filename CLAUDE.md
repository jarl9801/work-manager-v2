# CLAUDE.md — FiberOps Pro (Work Manager v2)

## What Is This?
PWA for tracking fiber optic installation projects — appointments, production, teams, certification.
Vanilla JS + IndexedDB, deployed as static site on GitHub Pages.

## Quick Start
```bash
cd ~/Dev/work-manager-v2
# No build step — open index.html or use a local server
# Deploy:
git add -A && git commit -m "description" && git push
# GitHub Pages auto-deploys from main
```

## Architecture
Single-page PWA, vanilla JavaScript, IndexedDB (`WorkManagerV2` database, version 2).

### 10 Views
Dashboard, Proyectos, Producción, Equipos, Clientes NE3, Citas WC, Citas GFP, Certificación, Hub, Settings

### Key Files
| File | Purpose |
|------|---------|
| `index.html` | Entry point, all HTML structure |
| `js/app.js` | Router, view switching |
| `js/db.js` | IndexedDB CRUD operations |
| `js/ne4citas.js` | WestConnect appointment management |

## External APIs
- **imasm (GFP):** `POST https://api-imasm.insytedeutschland.de/api/authorize` with `{usuario, password}`
- **Citas data:** `https://jarl9801.github.io/field-report/citas.json` (generated by LobsterOps cron every 15min)
- **GFP credentials:** stored in `localStorage` (`gfp_usuario`, `gfp_password`) — set in Settings → Credenciales

## ⚠️ CRITICAL

### Service Worker (sw.js)
**MUST use network-first strategy.** Cache-first caused stale content bugs that were painful to debug.
After ANY code change: **increment the cache version** (`wm-v2-cache-vN`) to bust client caches.

### Global Scope for Event Handlers
All `onclick` functions **must be on `window.*`** — HTML event handlers require global scope in this architecture.

### Deploy Checklist
1. Make changes
2. Increment `sw.js` cache version
3. `git add -A && git commit -m "..." && git push`
4. GitHub Pages auto-deploys (takes ~1 min)
